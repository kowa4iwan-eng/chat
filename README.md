# Shadow Chat — realtime messenger

Це Node.js застосунок з Express і Socket.IO. Повідомлення проходять через сервер і можуть доходити між різними пристроями.

## Що змінилося

- підказки з логінами і паролями прибрані з екрана входу;
- запуск за замовчуванням переведений на `server-memory.js`;
- файлова база більше не потрібна для стандартного запуску;
- дані зберігаються в пам’яті сервера;
- після перезапуску сервера користувачі, групи й повідомлення скидаються;
- додані аватарки користувачів;
- додані розширені адмін-настройки користувачів.

## Що вже є

- логін і пароль;
- серверна сесія після входу;
- адмін-панель;
- створення користувачів;
- редагування імені, логіна, пароля, ролі та аватарки;
- налаштування прав користувача;
- блокування та активація користувачів;
- створення груп;
- приватні чати;
- групові чати;
- онлайн-статус;
- смайли;
- фото і відео як прев’ю;
- мобільна адаптація;
- нові повідомлення приходять через Socket.IO.

## Запуск локально

Потрібен Node.js 18 або новіший.

```bash
npm install
npm start
```

Після запуску відкрий:

```text
http://localhost:3000
```

## Адмін-вхід

Краще задати адмін-логін і пароль через змінні середовища:

```bash
ADMIN_LOGIN=myadmin ADMIN_PASSWORD=mypassword npm start
```

Якщо `ADMIN_PASSWORD` не заданий, сервер створить тимчасовий пароль і покаже його в консолі запуску.

## Перевірка повідомлень

1. Відкрий сайт у двох браузерах або на телефоні й комп’ютері.
2. Увійди під різними користувачами.
3. Напиши повідомлення в приватному або груповому чаті.
4. Повідомлення має з’явитися одразу на другому пристрої.

## Важливо

GitHub Pages для цієї версії не підходить, бо він не запускає Node.js сервер. Для роботи онлайн потрібен VPS або Node.js hosting.

## Основні файли

- `server-memory.js` — основний сервер без файлової бази;
- `server.js` — старий варіант із файловим збереженням;
- `package.json` — залежності та запуск;
- `index.html` — сторінка;
- `style.css` — дизайн;
- `admin.css` — стилі аватарок та адмін-редагування;
- `script.js` — frontend.
